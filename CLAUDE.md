# Claude Code Instructions

## Development

- ALWAYS use TDD: write failing tests first, then implement code to make them pass.
- This applies to new features, bug fixes, and refactors.
- ALWAYS run all available tests prior to concluding a code update is complete.

## Commit Messages

- List out actual code changes (files added, modules implemented, features built).
- Do NOT reference planning documents, spec docs, or any files not tracked by git.
- Do NOT reference phase numbers or planning terminology.

## README

- Always include a README.md file.
- Use a TOC with the README.md file.
- Keep the project README.md file up-to-date after code changes that affect:
    - Information regarding installation and configuration
    - Information related to running the app
    - Key architectural information important to understanding how the app works
